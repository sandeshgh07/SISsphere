# Contributing

Thanks for contributing.

## Before you start

- Search existing issues and pull requests first.
- For large changes, open an issue before writing code.
- Be respectful and constructive in discussion.

## Development flow

1. Fork the repository.
2. Create a branch: `git checkout -b feat/short-description`
3. Make your changes.
4. Add or update tests and docs.
5. Run the test and lint commands.
6. Commit with a clear message.
7. Open a pull request.

## Pull request checklist

- Code builds locally.
- Tests pass.
- Documentation is updated.
- No secrets or private data were added.
- Public APIs or behavior changes are explained.

## Commit style

Use short, descriptive commits. Conventional Commits are recommended but optional.

## Code review

Maintainers may request changes before merge. Please keep pull requests focused and reasonably small.
